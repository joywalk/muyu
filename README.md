# 电子木鱼 - 微信小程序

极简禅意风格的微信小程序，点击木鱼获得功德累积。

## 项目结构

```
muyu/
├── app.js              # 小程序入口
├── app.json            # 小程序配置
├── app.wxss            # 全局样式
├── sitemap.json        # 站点地图
├── pages/
│   ├── index/          # 首页
│   │   ├── index.js    # 首页逻辑
│   │   ├── index.wxml  # 首页页面
│   │   └── index.wxss  # 首页样式
│   └── settings/       # 设置页
│       ├── settings.js    # 设置逻辑
│       ├── settings.wxml  # 设置页面
│       └── settings.wxss  # 设置样式
├── images/             # 图片资源目录
│   └── fish.png        # 木鱼图片（需要添加）
├── audio/              # 音频资源目录
│   └── fish-sound.mp3  # 敲击音效（需要添加）
└── utils/              # 工具函数目录
```

## 核心功能

### 1. 核心敲击功能
- ✅ 点击木鱼播放敲击音效（支持快速连点）
- ✅ 木鱼视觉反馈（缩放/抖动动画，0.1-0.2s）
- ✅ "+1" 浮动动画（向上飘 + 渐隐 + 随机偏移）
- ✅ 功德数值实时递增
- ✅ 轻微震动反馈（可配置）

### 2. 音频实例池
- 使用 5 个 InnerAudioContext 实例轮询
- 解决连点时的音效卡顿问题
- 每次点击使用不同实例，避免冲突

### 3. 功德数据管理
- ✅ 本地存储功德值（持久化）
- ✅ 应用重启后自动恢复
- ✅ 提供清零功能（二次确认）

### 4. 设置功能
- ✅ 音效开关（默认开启）
- ✅ 震动开关（默认开启）
- ✅ 每次增量设置（+1/+2/+3/+5/+10，默认+1）
- ✅ 动画显示开关（默认显示）

## 技术方案

### 音频实例池
```javascript
// 创建 5 个音频实例轮询使用
const poolSize = 5;
for (let i = 0; i < poolSize; i++) {
  const audio = wx.createInnerAudioContext();
  audio.src = '/audio/fish-sound.mp3';
  audioPool.push(audio);
}

// 播放时轮询
pool[poolIndex].play();
poolIndex = (poolIndex + 1) % pool.length;
```

### +1 动画管理
```javascript
// 限制最大同时显示 12 个
if (plusAnimations.length > 12) {
  plusAnimations.shift();
}

// 1秒后自动移除
setTimeout(() => {
  // 移除动画
}, 1000);
```

## 需要准备的资源

### 1. 木鱼图片
- 位置：`images/fish.png`
- 建议尺寸：200x200px
- 格式：PNG（推荐透明背景）
- 风格：简约白色木鱼图标

### 2. 敲击音效
- 位置：`audio/fish-sound.mp3`
- 建议时长：0.3-0.5秒
- 格式：MP3
- 音质：清晰短促的敲击声

## 使用方法

### 1. 准备资源
将木鱼图片和音效文件放入对应目录：
- 木鱼图片 → `images/fish.png`
- 敲击音效 → `audio/fish-sound.mp3`

### 2. 导入微信开发者工具
1. 打开微信开发者工具
2. 选择"导入项目"
3. 选择 `muyu` 目录
4. 填写 AppID（测试号或正式号）
5. 点击"导入"

### 3. 运行测试
- 点击"编译"按钮
- 在模拟器或真机中测试
- 测试快速连点是否流畅

## 注意事项

1. 首次播放音频需要用户交互触发
2. 音频文件大小建议控制在 100KB 以内
3. 图片文件建议使用 webp 格式压缩
4. 功德值存储在本地，清除缓存会丢失数据

## 开发建议

1. **音频文件**：建议使用短促、清脆的敲击声
2. **木鱼图片**：简约风格，白色背景更佳
3. **动画时长**：建议控制在 0.2s 以内，避免卡顿
4. **音频池大小**：可根据实际连点速度调整（建议 5-10 个）
